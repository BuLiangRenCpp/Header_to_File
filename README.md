# Header_to_File

### 介绍
1. 主要功能：根据头文件对函数或者类的成员函数声明，自动生成源文件的对应定义语句的命令行简易工具。 (以下此功能统一称为 " htf " )

2. 优点：方便在我们声明完头文件后，不用重写一遍或者copy，直接一条指令生成基本代码框架，从而专注于代码实现

3. 功能详情及限制：       
    (1) 仅使用 C++语法 的头文件     
    (2) 目前支持：  
    **a.** 一般函数的声明转定义
    ```cpp
    int add(int, int);   
    --->  
    int add(int, int)
    {

    }     
    ```

    **b.** 类的成员函数转定义
    ```cpp
    class A {
        int* add(vector<int>, string s = "sjifd");
    };
    --->  
    int* A::add(vector<int>, string s)
    {

    }
    ```
    **c.** namespace 中的 a、b
    ```cpp
    int fic();
    namespace A {
        int A();
        namespace B {
            int B();
        }
    }
    ---> 
    int fic()
    {

    }

    namespace A {
        int A()
        {

        }

        namespace B {
            int B
            {

            }

        }

    }
    ```

    **d.** htf 单文件   
    ```
    将指定路径的头文件在给定的目录下(默认与头文件路径相同)生成对应的源文件
    ```
    **e.** htf 文件夹   
    ```
    将指定目录下的所有头文件在给定目录下(默认与头文件目录相同)生成对应的头文件
    ```
4. 其他功能待扩展 ......

### 命令行使用说明    

>         简易版提供的指令与详细版的指令都能完成所有工作，只不过详细版更为灵活而已，如果你没兴趣看详细版，简易版中的常用指令基本够用 ( 只要严格遵循指令格式即可，简易版相当于详细版的默认格式 )。

#### 1. 简易版 

1. 约定：   
    (1) [ ]：括号中的内容可有可无   
    (2) 英文是程序所指定的指令，中文请对照原文意思输入对应的内容    
    (3) 目录：文件夹对应的绝对路径，如 " D:/dirs " or " D:/dirs/ "    
    (4) 路径：文件对应的绝对路径，如 "D:/dirs/file.h"       
    (5) 文件名：不包含扩展名的文件 (头文件使用 .h，源文件使用 .cpp作为扩展名，所以不必输入扩展名)                      
    (6) 源文件默认生成模式: 生成的源文件在头文件所在目录，且源文件名与头文件名相同，只是扩展名不同  
    (7) 默认目录：当前工作目录

2. 注意：   
    (1) 目录(或路径)中有空格的必须使用英文双引号 "" 包含起来，无空格可用可不用   
    (2) 当输入与指令或者参数冲突时，请用 "" 包含起来        
    (3) 按行输入，输入完毕后点击回车即可执行指令 

3. 指令格式：
    
    (1) 查看默认目录 
    ```
    指令：      ld

    说明：   list default_dir
    ```
    
    **(2) htf 单文件 ( 源文件默认生成模式 )**
    ```
    指令:    htf   头文件的路径或文件名    

    说明：
        1. 输入的是文件路径，则使用所输入的路径
        2. 输入的是文件名(假设是 header)，则将使用 默认目录 下的文件
        3. 相对路径写法：
            同终端写法相同 (如 ../parent_dir、./current_dir)
    ```
    
    **(3) htf 目录下的所有头文件 ( 源文件默认生成模式 )**
    ```
    指令:    htf   目录

    说明:   
        将 目录 下的所有头文件转为对应的源文件，源文件保存在默认输出目录下
    ``` 

    (6) 退出程序：
    ```
    指令：    q
    ```


#### 2. 详细版 ( 在简易版中提到的不在赘述 )    

1. 约定：   
    (1) ins：instruction，命令行指令    
    (2) arg：argameter，参数，必须以 - 开头    
    (3) con：content，除ins、arg外的内容，大多是路径或者目录

2. 注意：       
    (1) ins、-arg、con之间以空格隔开，-arg与con可以连在一起    

3. ins:         
    (1) ld：list default_dir，查看默认目录  
    (2) htf：header_to_file     
    (3) q：quit，终止程序   

4. arg:       

- 有参型： ( arg后可接自己的参数，arg与其后所跟的参数是一个整体，以空格相隔 )   
    >  在输入时应加上 -       

    - i：input，读取目录(或路径)，后跟头文件目录(或路径)   
    ```
    1. -i D:\dirs      表示 指定读取目录为 D:\dirs，
    2. -i D:\dirs\file      表示 指定读取路径为 D:\dirs\file.h  
    ```
    - o：output, 输出目录(或路径)，后跟生成的源文件目录(或路径)   

- 无参型： 
    >  在书写时应加上 - 

    - f: force，强迫执行，不提示信息
    > 为了避免程序输出的文件错误地覆盖原有文件，将是否覆盖文件的要求给用户自行决定

> 注： 不同的 arg整体 可相互调换在指令中的顺序

5. 指令格式：

    >     基本格式："ins -arg content ..."

    (1) htf 单文件 ( 默认 htf 第一个是 -i，第二个是 -o ) 
    ```
    指令：   htf  [-i 头文件的路径或文件名  [-o 源文件的目录]] 

    说明：(default_dir = " D:\dirs\ ")
        1. 如果只给出头文件名，则使用 idefault_dir目录下的对应的头文件名相同的文件路径 
        2. -i -o 如果只写一个，则自动推导另外一个
    ```     
    

    (2) htf 目录下的所有头文件      
    >  注：其默认格式以及参数格式与 "htf 单文件" 的默认格式与参数格式一样，只是输入的是目录  

    ```
    指令：  htf  头文件目录或者-d   [源文件目录或者-D]    [-f]
            htf   -i 头文件目录或者 id   -o 源文件目录或者 od

    说明：
        同 htf 单文件
    ```


#### 注意 
- 此工具默认目录是当前工作目录，因此将其加入环境变量然后在终端使用较为方便
- 在终端使用此工具后，需要退出才能正常使用电脑自带终端的功能，该工具不提供终端的改变目录 ( cd ) 的功能
